A Database Management System provides the mechanism to store and retrieve data.
There are many kinds of DBMS available in market

  a) Relational Databases 	- SQL Databases
  b) Non-Relational Databases 	- NoSQL Databases

SQL DBs and NonSQL DBs differ in terms of data retrieval, distribution, storage and processing

How data will be managed in SQL / Relational DBs?
 -> table of rows and columns - with a strict structure and clear dependencies
 -> proper structure and clear relationships
 -> Oracle, MySQL, SQL Server, PostgeSQL, DB2 etc


How data will be managed in NoSQL?
 -> Non-Relational Databases are not only limited to a Table structure - NoSQL.
 -> NoSQL databases are document oriented - in form of documents
 -> we can work with non-structured data like articles, photos, videos etc are collected as a single document
 -> MongoDB, Redis, Casssandra, HBase etc
 -> documents can be JSON documents, wide-column stores, key-value pairs etc

When to go for NoSQL Databases?

-> when you want to store and retrieve huge amount of data - big data
-> the relationship between the data that you store is not that important
-> that data is not structured and changing over time
-> constraints and joins support is not require at DB level
-> the data is growing continuously and you need to scale the database regularly to handle the data

Vertical Scaling	- resource increase - RAM increase
Horizontal Scaling	- add new server


What is MongoDB?

-> MongoDB is a popular Non-Relational Database - to deal with huge amount of un-structured data
-> MongoDB is open source, cross-platform, document-oriented database
-> MongoDB is written in C++
-> MongoDB  provides high performance, high reliability, and automatic scaling
-> MongoDB  is developed by a US based company 10gen (2007) - name changed to MongoDB, Inc (2013)



RDBMS Vs MongoDB

1) RDBMS is having Relational Database. MongoDB has Non-Relational Database
2) In RDBMS we need to design DB Tables then only we can do the coding. In MongoDB we can directly start the coding 
3) RDBMS supports SQL language. MongoDB supports SQL as well as JSON Query Language
4) RDBMS is Table based. MongoDB is key-value based
5) RDBMS is row/record based. MongoDB is document based
6) RDBMS is column based. MongoDB is field based
7) RDBMS is Vertically Scalable. MongoDB is Horizontally Scalable
8) RDBMS process data slowly. MongoDB process data fastly

$$$--- MongoDB UPDATE Documents ---$$$

In MongoDB, update() method is used to update or modify the existing documents of a collection.  

Syntax: db.COLLECTION_NAME.update(SELECTION_CRITERIA, UPDATED_DATA)

Collection.update() is deprecated. Use updateOne(), updateMany(), or bulkWrite().

We can use $set and $inc operators to update any field in MongoDB.  
The $set operator will set the newly specified value while the $inc operator will increase the value by a specified value.

Update an existing document

-- update the existing course "Java" to "Python"
> db.courses.update({'course': 'Java'}, {$set: {'course' :'Python}})   -- updates only the first match
> db.courses.updateMany({'course': 'Java'}, {$set: {'course': 'Python'}})

> db.courses.find({'course': 'Python'})
> db.courses.find({'course': 'Python'}).count()

//for case sensitive search - we should use regex		> db.courses.find({'course': /^python$/i}).count()
> db.courses.find({course: /^python$/i}).count()

> db.courses.updateMany({'course': /^java$/i}, {$set: {'course': 'Python'}})


$inc usage - operator will increase the value by a specified value.
> db.employees.updateOne({eid: 101}, {$inc: {esal: 100}})

###--- MongoDB DELETE Documents ---###

remove(deletion criteria, JustOne) method works on 2 parameters
1) deletion criteria : with the use of its syntax , we can remove the document from the collection
2) JustOne - it removes only one document when set to true or 1

Syntax: db.collection_name.remove(DELETION_CRITERIA)

Remove All Documents - pass an empty query document {} to remove() method . 
The remove() method does not remove the indexes

> db.courses.remove({})
Collection.remove() is deprecated. Use deleteOne(), deleteMany(), findOneAndDelete(), or bulkWrite().


--- Remove All Documents that match a condition ----
> db.courses.remove({'type' : 'Programming Language'})

--- Remove a Single Document that match a condition ----

> db.courses.remove({"category" : 'Programming language'}, 1) //removes only on1
> db.courses.deleteMany({category: 'Programming Language'})
> db.courses.deleteMany({category: /^Programming Language$/i})

========  Indexing in MongoDB =============

https://www.geeksforgeeks.org/indexing-in-mongodb/?ref=ml_lbp

Indexing in MongoDB is a crucial feature that enhances query processing efficiency. 

Indexes are used to increase the speed of retrieval of data when we have large chunks of documents in a collection
ex: book index?

Without indexing, MongoDB must scan every document in a collection to retrieve the matching documents and leading to slower query performance. 

- by default - every collection will have an Index on "_id" key		-- check in MongoDB Compass

> db.employees.getIndexes()
> db.employees.getIndexes().forEach(function(ind){print(ind)})
> db.employees.getIndexes().forEach(function(ind){print(ind.name)})

db.COLLECTION_NAME.createIndex({KEY:1}) 

> db.employees.createIndex({"name": 1})  //ascending
> db.employees.ensureIndex({"empId": -1}) //descending

> db.employees.dropIndex('name_1')

===== Aggregation Framework in MongoDB ==========

The purpose of MongoDB's Aggregation Framework is to design a pipeline consisting of multiple stages for processing documents. You start with your collection's data and after each stage of the pipeline you are closer to the end result which will be the desired documents.
